# Analogical Learning-Based Wireless Localization (ALLoc)

Zirui Chen, Zhaoyang Zhang, Ziqing Xing, Ridong Li, Zhaohui Yang, Richeng Jin, Chongwen Huang, Yuzhi Yang and Mérouane Debbah

------

This repository is the official implementation of paper [Analogical Learning for Cross-Scenario Generalization: Framework and Application to Intelligent Localization](https://arxiv.org/abs/2504.08811).

**Note: The latest version of this repository is a reimplementation of the original version, featuring clearer logic and improved efficiency. The original implementation has been tagged as `v1` and can be accessed [here](https://github.com/ziruichen-research/ALLoc/tree/v1).**

## Overview of This Work

This work proposes a new framework called ***Analogical Learning*** (AL). It enables direct adaptation to diverse scattering environments and system configurations without requiring additional training, offering a new technical system with cross-scenario generalization for intelligent wireless localization and various other wireless AI tasks. This breakthrough stems from a key insight into the relativity of numerical feature representations: isolated numerical values generally cannot convey complete meaning unless combined with corresponding reference frame information. The lack of ability to aware reference system information restricts existing wireless AI methods (e.g., data-to-label learning) to generalizing only within specific underlying reference systems, resulting in poor cross-scenario reusability and high sensitivity to dynamic environmental changes.

Therefore, to complement reference frame information, AL first embeds known data-label pairs from the current scenario as reference objects in the training and inference process, resembling the in-context learning (ICL) in the expression form. The most notable innovation lies in the development of ***Mateformer***, inspired by the insight of relativity. This network designs an advanced multi-layer analogical mechanism, enabling the interpretation of feature information through multi-space relativity and fully leveraging reference frame information. The application of AL to wireless localization demonstrates its superiority: it achieves **state-of-the-art accuracy**, **stable transferability**, and **seamless adaptation to new scenarios without any tuning**, outperforming conventional methods by nearly two orders of magnitude in precision. This expansion of wireless intelligence in multi-scenario dimension marks an important step toward [wireless big AI models](https://ieeexplore.ieee.org/document/10579546).

## Usages

### Dataset

The experiments in O1, O1B, and MO1 scenarios are based on DeepMIMO V1, while the experiments in RO1 and O2 scenarios are based on DeepMIMO V2. The RO1 data is generated by adding rain attenuation to the O1_60 dataset according to [ITU-R P.838-3](https://www.itu.int/rec/r-rec-p.838-3-200503-i). We have reorganized the datasets into HDF5 format, which can be downloaded via the following links.

*Google Driven*: https://drive.google.com/drive/folders/183kxLUp7nJJxgZ8zQ7lJT80xRmO5HrZT?usp=drive_link

*Baidu Netdisk*: https://pan.baidu.com/s/1IP47KTdbG0E4utnpKERfVA?pwd=3xbu

After the download is complete, please place all the compressed files in the same directory, then run the following commands.

```shell
cat dataset_deepmimo.tar.gz.* | tar xvzf -
cat dataset_deepmimo_rain.tar.gz.* | tar xvzf -
cat dataset_deepmimo_road.tar.gz.* | tar xvzf -
```

This will create three subdirectories: `dataset_deepmimo/`, `dataset_deepmimo_rain/`, and `dataset_deepmimo_road/`. Please move them into the directory of this repository.

`dataset_deepmimo/` contains the data files required for experiments in O1, O1B, and MO1 scenarios, while `dataset_deepmimo_rain/` and `dataset_deepmimo_road/` contain the data files required for experiments in RO1 and O2 scenarios, respectively.

The data files in these subdirectories require 2.5GB, 4.7GB, and 16GB of storage space, respectively. To improve training efficiency, we provide the *gen_seq_datasets* parameter in the code; when it is set to True, each training dataset will be preprocessed and expanded to a much larger size during the ALLoc training process. In this case, we recommend ensuring approximately 100GB of available storage space before each training task in O1/O1B/MO1 scenarios, 320GB of available storage space before training in RO1 scenario, and 1.1TB of available storage space before training in O2 scenario. For more details, please refer to the description of the *gen_seq_datasets* parameter in `README.doc.md`.

### Training and Testing

In our newly implemented version, all experiments run the same code, `main.py`, but with different configuration parameters. Specifically, each experiment is configured using a source configuration file specified via the `--conf` option; additionally, parameters from the source configuration file can be temporarily modified for the current experiment via the `--mod` option.

Each experiment is identified by the *exp_dir* and *task* parameters. When a new experiment is launched, a copy of the source configuration file will be created in the *exp_dir* directory and renamed as `train.yaml` (if *task* is 'train') or `test.yaml` (if *task* is 'test'). Parameter modifications made via the `--mod` option will be applied to the copied configuration file, while the source configuration file remains unchanged.

Configuration file templates for both training and testing are provided in the `conf/` directory. Therefore, a convenient way to configure an experiment is to specify a template as the source configuration file using the `--conf` option, and modifying certain parameters in the copied configuration file using the `--mod` option.

The following example trains AD_CNN in DeepMIMO O1 scenario, making use of the `conf/train.yaml` template and modifying the following parameters: *exp_dir*, *data_dirs*, *scheme*, *num_train_data*, and *init_lr*.

```shell
python main.py --conf=./conf/train.yaml --mod exp_dir=./exp/o1_40k/ad_cnn/ \
    data_dirs=[./dataset_deepmimo/o1/] \
    scheme=ad_cnn num_train_data=40000 init_lr=0.001
```

If the *exp_dir* and *task* parameters match those of a previous experiment, the code will use that experiment's existing configuration file and ignore any new configurations specified via the `--conf` or `--mod` options. This is particularly useful for resuming training from the latest checkpoint.

```shell
python main.py --conf=./exp/o1_40k/ad_cnn/train.yaml
# Equivalent to
python main.py --conf=./conf/train.yaml --mod exp_dir=./exp/o1_40k/ad_cnn/
```

We have included many of the configuration files used in our experiments in the `exp/` directory. For more usage examples and detailed descriptions of the configuration parameters, please refer to `README.doc.md`.

In our experiments, all models can be trained using a single GPU with 32GB of memory (both the model and data are based on float32). If the device has insufficient memory, the *batch_size* parameter can be appropriately reduced. Additionally, we have provided all model files in this repository, so testing can be performed directly without training.
